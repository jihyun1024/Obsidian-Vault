### 인공지능기술 개념
- 규칙 기반 방법론
	- 전문가가 알고리즘을 가지고 사전에 코딩 > 데이터를 입력해 결과를 얻음
	- 짜장면을 시키면 요리사가 직접 조리해서 배민이 배달하는 것과 비슷
- 기계학습 방법론
	- 컴퓨터에게 데이터를 학습시켜 패턴과 규칙을 컴퓨터가 스스로 학습해 그 데이터와 결과로부터 규칙을 뽑아내는 구조 (데이터 주도 패러다임)
	- 짜장면의 레시피 특성을 학습하는 것과 비슷

- 추세
	- 비약적인 발전 중 (ANI 또는 AGI)
	- 발전 수준: ANI > AGI > ASI
		- ANI: 특정 분야에서 사람과 비슷한 수준
		- AGI: 다양한 분야에서 대체적으로 사람보다 더 나은 수준
		- ASI: 모든 분야에서 사람보다 월등히 나은 수준
	- 인공지능과 같이 공존 > LLM이 짠 코드를 판별할 수 있는 역량 정도는 가지고 있어야 함

- 인공지능 기술 분류
	- 머신러닝 vs 딥러닝
	- 지도 학습, 비지도 학습, 강화 학습
- 학습 모델과 알고리즘
	- ML 모델: ML에 관련된 수식 또는 통계 분포
	- ML 알고리즘: ML 모델을 학습시키기 위한 알고리즘 기법 (프로그래밍 관점)
		- ML 알고리즘은 ML 모델을 만들어내는 기법
- 수학적 관점
	- 기본 모형 (수학적 모형)
	$$y = f(x)$$
		- $x$: 입력, $y$: 출력, $f()$: 함수
		- 머신 러닝은 우리가 찾고자 하는 $f()$를 제공
	- 모델
		- 입력과 출력의 상관 관계를 1차 선형 방정식으로 표현
		- 학습: 해당 선형 방정식을 찾는 과정 
- 인공지능 학습 
	- 학습 단계 (Training Phase): 주어진 데이터를 머신 러닝 알고리즘에 적용해 학습
	- 예측 단계 (Prediction Phase): 학습 단계에서 생성된 모형에 데이터를 적용해 예측
		- 초기 상태는 전부 동일하나, 각각의 핸드폰의 기종이 같더라도 용도에 따라 언제 쓰는지, 사용자의 숙련도가 어느 정도인지 등등의 결과값이 달라지듯이 어떤 데이터를 학습시켰는지에 따라 같은 모델이라도 성능이 다를 수 있음
- 개발 관점 Work Flow
	1. 문제 이해
		- 주어진 도메인 문제와 목적을 이해
	2. 탐색적 데이터 분석
		- 데이터에 대한 전처리를 수행하는 과정
			- 전처리: 문제 데이터를 숫자, 바이너리 등으로 변화시키는 과정
		- 데이터에 대한 이해 또한 필수
	3. 베이스라인 모델 구축
		- 기본 학습 모델을 만드는 과정
	4. 성능 개선
		- 방법: 하나의 모델을 개선 또는 교체, 파라미터 최적화, 피처 엔지니어링 등등
