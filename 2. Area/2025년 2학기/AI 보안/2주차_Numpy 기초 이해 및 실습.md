### Feature 정의 및 사례
Feature는 데이터의 특성을 표현하는 것으로, 머신러닝이나 데이터 분석에 사용되는 개별 독립 변수를 의미한다. 모델을 구성할 때 데이터가 가장 큰 영향을 주는데, 모델은 입출력의 함수 형태로 다음과 같이 표현된다. 
![[Pasted image 20250912045114.png]]
예를 들어 영화관에서 관람객의 수를 웹사이트의 좋아요 개수로 예측할 수 있다고 가정하면, 간단하게 한 종류의 $x$값, 즉 좋아요 개수로 $y$값, 즉 관람객 수를 예측할 수 있다. 

조금 더 복잡하게는 다음과 같이 적용할 수 있다. 
![[Pasted image 20250912045438.png]]
이 경우 독립변수는 13개가 되며, 이 여러 개의 독립변수로 종속변수를 예측할 수 있다. 

데이터 테이블은 데이터를 행렬 테이블로 표현한 것으로, Feature는 데이터 테이블 상에서 하나의 열에 해당하며 데이터 인스턴스는 하나의 데이터, 즉 하나의 행에 해당한다. 
이 하나의 행, 즉 튜플 값은 열 벡터로 표현하며 가중치도 열 벡터로 표현하고, 위에 있던 수식도 또한 선형대수 표기법으로 표현할 수 있다. 
![[Pasted image 20250912045915.png]]
이 때 $W^T X$는 벡터의 내적(dot product)으로, 열 벡터끼리 같은 위치에 있는 값들을 곱해 모두 더하는 연산이다. 

이 과정에서 차원의 저주라는 개념이 나오며 이는 피처의 개수 = 차원이 커질수록 관련 데이터를 많이 확보해야 모델의 성능 또한 확보되는데, 고려해야 할 것이 많아질수록 데이터를 표현해야 하는 공간이 지속적으로 늘어나게 되고 이에 대한 처리가 어려워진다. 
이는 데이터 처리 속도와 메모리 공간에 대한 문제를 유발한다. 

예를 들어, 외식을 하려 할 때 2명만 간다면 먹을 음식을 금방 고를 수 있지만 20명이 가게 되면 각자의 선호도가 전부 다를 수 있기 때문에 먹을 음식을 금방 고르지 못하는 문제가 생긴다. 
이는 차원의 저주 개념과 비슷하다고 볼 수 있다. 

이에 대한 해결 방법은 **선형방정식으로 풀지 않고 비선형방정식으로 푸는 방법** 등이 있다. 

### 학습 데이터 유형 이해
데이터를 모델에 학습시킬 때, 무작정 데이터를 모델에 집어넣으면 모델은 해당 데이터를 제대로 처리하지 못한다. 이를 위해서 학습할 데이터의 유형을 이해해야 한다. 

- 수치형 데이터
	- 수치적 의미를 가지고, 숫자 형태로 나타나는 데이터 **(사칙연산 가능)**
	- 수치형 안에서도 연속형, 이산형 데이터로 나누어진다.
		- 연속형: 실수 or 정수 값으로, 온도, 자동차 속도, 키, 몸무게 등
		- 이산형: 정수로 딱 떨어지는 값으로, 우편번호, 사고 건수, 성별(문자형 텍스트) 등
	- ==문자 형태 값은 반드시 텍스트 값을 숫자 형태로 바꾸어서 변환해 할당해야 한다.==
- 범주형 데이터
	- 몇 개의 항목으로 분류 가능한 데이터 **(사칙연산 불가능)**
	- 범주형 안에서도 순서형, 명목형 데이터로 나누어진다. 
		- 순서형: 데이터 간의 순서 or 순위가 존재하는 값으로, 크기, 학점, 메달 색 등
		- 명목형: 순서나 순위가 없는 값으로, 성별, 우편번호 등 카테고리로 분류 가능

### Numpy 기본 개념
Numpy는 다차원 배열을 효과적으로 처리해서 강력한 데이터 처리를 하기 위한 도구로, 벡터, 행렬 등 수치 연산을 수행하는 선형대수 연산들을 효율적으로 처리하는 [[Python]]의 라이브러리이다. 

Numpy에서 배열의 객체는 ndarray or array라고 하며 이 때 "nd"의 뜻은 n차원 dimension의 줄임말이다. 

![[Pasted image 20250912052022.png]]
이 과정은 수학적으로 표현된 수식을 Python에서 Numpy를 이용해 변환하는 계산이다. 
이 상황에서, 좋은 코드는 똑같은 작업을 수행하더라도 더 적은 양의 코드로 수행할 수 있는 코드이며, 이것을 이 사진이 잘 보여준다 할 수 있다. 

해당 사진에서 $MeanSquareError$식과 `error`공식은 수학적으로 똑같으며, 다른 점이라 해 봤자 더 간단하게 표현해서 더 좋은 코드로 변환한 것 밖에 없고, 이 두 공식은 오차를 줄이기 위한, 즉 최적화를 위한 계산이라는 것이 공통점이다. 

인공지능 모델에 Input되는 데이터는 크게 3가지로 나눌 수 있다. 
1. 자연어(언어) 데이터
2. 이미지 데이터
3. 음성 데이터
이 3가지 중 가장 처리가 쉬운 것은 의외로 이미지 데이터로, RGB 값으로 2차원 배열화되어 표현되고, 이외에는 별다른 점이 없기 때문에 의외로 심플하다. 
그러나, 자연어의 경우에는 사람마다, 언어마다 주어와 동사의 위치가 달라 그 둘의 연관성을 파악하기 어렵고 이를 분석하기 위해서는 빈도 분석 기반으로 연관성 분석, 숫자 배열 형태로 만들어 3차원 배열로 전처리가 필요하다. 
이 3가지에서 가장 복잡한 것은 음성 데이터로 음성의 주파수 파형을 쪼개 숫자화하고, 이후 자연어 처리까지 이어서 해야 해서 매우매우 복잡하다. 

이 밑부터는 교재에 나온 그대로 옮겨 적을 것이다. 

---
- 넘파이 배열 객체(numpy.ndarray)의 속성값
	- numpy: 파이썬의 수치 계산 라이브러리 모듈 이름
	- numpy.ndarray 클래스의 인스턴스 변수
		- numpy: 파이썬의 수치 계산 라이브러리 모듈 이름
		- ndarray: numpy 모듈에 정의된 다차원 배열의 클래스
	- 역할: 배열의 차원 크기(행/열 등)를 튜플 형태로 제공 (읽기 전용)

**ndarray.shape**
- 설명: 배열의 각 축(axis)의 크기를 튜플 형태로 반환
- 형식: `(dim0, dim1, ..., dimN)`
**ndarray.ndim**
- 설명: 배열의 차원 수 = 축의 개수를 반환
- 형식: 정수 (`int`)
**ndarray.dtype**
- 설명: 배열 요소의 데이터 타입을 나타냄 (예: `int32`, `float64` 등)
- 비고: `type()` 함수는 객체의 전체 타입 확인, `dtype()`은 요소 단위의 데이터 타입 확인
**ndarray.itemsize**
- 설명: 배열의 각 요소가 메모리에서 차지하는 크기 (bytes 단위)
- 형식: 정수 (`int`)
**ndarray.size**
- 설명: 배열 전체 요소의 개수
- 계산식: `shape`의 각 값의 곱과 동일

![[Pasted image 20250912053814.png]]

**예시 코드**
```python
import numpy as np

a = np.array([1, 2, 3, 4, 5, 6]) # 1차원 배열 생성

print(a.shape) # (6,) --> 열은 1개라서 따로 숫자로 나오지 않음
print(a.ndim) # 1
print(a.dtype) # int64로, 배열 요소 단위의 자료형
print(type(a)) # <class 'numpy.ndarray'>로, 배열 전체의 자료형
print(a.itemsize) # 8 --> 64bit = 8byte
print(a.size) # 6 --> 배열의 전체 원소 수
```
