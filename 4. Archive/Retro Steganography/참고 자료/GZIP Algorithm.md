gzip은 GNU zip의 약자로, ZIP과 같이 [[DEFLATE 압축 알고리즘]]을 따르지만, 여러 파일을 하나의 파일로 압축하는 옵션이 없다는 점에서 차이가 난다.
따라서 여러 파일 또는 여러 디렉터리를 하나의 파일로 압축하기 위해 보통 Tar 알고리즘과 같이 사용하여 .tar.gz로 압축된 파일의 경우 zip과 기본적인 압축 알고리즘은 같지만 더 용량이 적다. 이는 .tar.gz의 경우 서로 다른 파일에서 중복되는 부분을 압축시킬 수 있기 때문이다. 

--- 
### 구조
GZIP은 단순한 압축 포맷이라기보다는 여러 정보를 담고 있는 컨테이너 형식으로 저장된다. 
기본 구조는 다음과 같다. 
1. Header (10 바이트 이상)
	* GZIP 파일임을 나타내는 magic: `1F 8B
	* 압축 방식 ([[DEFLATE 압축 알고리즘]]은 `08`)
	* 타임스탬프, OS, 압축 플래그 등 다양한 정보
2. Compressed Data
	* [[DEFLATE 압축 알고리즘]]을 사용하여 압축된 데이터
3. Optional Trailer
	* [[CRC Algorithm|CRC32]] (압축되기 전의 원래 데이터의 체크섬)
	* 원래 데이터의 크기
