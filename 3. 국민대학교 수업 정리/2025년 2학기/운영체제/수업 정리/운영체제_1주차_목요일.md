### 컴퓨터 하드웨어
컴퓨터 하드웨어는 세 가지로 나누어 진다. 
- 프로세서: CPU, GPU 등
- 메모리: 주 기억장치, 보조 기억장치 등
- 주변장치: 키보드/마우스, 모니터, 프린터, 네트워크 모뎀 등

이 중 우리가 집중해서 알아볼 것은 크게 다섯 가지로, **CPU, 주 기억장치, 보조 기억장치, 키보드/마우스, 모니터/프린터**이다.

### 프로세서
프로세서는 컴퓨터의 두뇌와 같은 것으로, 컴퓨터의 모든 장치의 동작을 제어한다.
CPU는 메모리에 저장된 명령어를 읽어 들이고, 해석하고 실행하는 부품으로 ALU(Arithmetic Logic Unit)와 제어장치, 여러 개의 레지스터로 구성되어 있다. 

**ALU**는 일종의 계산기로, 계산 명령만을 수행하며 **레지스터**는 CPU 내부의 작은 저장장치로 프로세서가 사용할 데이터를 저장하며 컴퓨터에서 가장 빠른 메모리이다. 또한 **제어장치**는 제어 신호를 내보내고 명령어를 해석하는 장치이다. 

예를 들어 100과 120을 더하는 명령어가 있다고 가정해 보자. 
그러면 처음 제어장치가 1번지에 있는 메모리를 읽어 3번지와 4번지에 있는 데이터를 가져오도록 한다. ![[Pasted image 20250913184502.png]]
그 후 피연산자를 저장하는 레지스터에 3번지와 4번지에 각각 있는 120과 100을 저장하면 ALU가 해당 계산을 실행하며 그 결과는 다른 레지스터에 저장된다. 
![[Pasted image 20250913184626.png]]
거기서 멈추지 않고 제어장치가 2번지에 있는 "연산 결과를 저장하라"는 명령어를 읽기 위해 제어 신호를 보내 메시지를 읽고 해당 메시지를 "더하라, 3번지와 4번지를" 명령어가 있던 레지스터에 저장한다. (편의상, 제어 신호를 보내는 부분은 생략했다.)![[Pasted image 20250913184901.png]]
그러면 메모리에 계산 결과를 저장해야겠다고 제어 장치가 판단해 아직 사용하지 않은 5번지에 제어 신호를 보내서 계산 결과값인 220을 해당 번지에 저장한다. 
![[Pasted image 20250913185232.png]]

### 운영체제와 프로세서
운영체제는 컴퓨터의 주요 부품 중 하나인 프로세서에도 영향을 주는데, 프로세서는 **프로세스**를 실행하는 역할을 하지만 해당 단일 프로세스가 너무 길게 실행되면 나머지 병렬로 실행할(것처럼 보일) 프로세스가 실행될 틈이 없기 때문에 각각의 프로세스를 관리한다. 

첫 번째로는 프로세스의 생성과 관리를 수행하며, ![[Pasted image 20250913185603.png|550x400]]
두 번째로는 방금 설명했던 프로그램의 프로세서 사용 시간을 관리하는데, 이를 **CPU 스케줄링**이라 한다. 이는 다양한 프로그램의 사용 시간 조율을 위해 필요하다. ![[Pasted image 20250913185726.png]]

**※ 프로세스와 프로그램의 차이:** 프로세스는 OS가 메모리 등의 자원을 할당해 준 '실행 중인 프로그램' 이며, 프로그램은 아직 실행 중은 아니지만 실행 할 수 있는 '정적인 상태'의 파일이다. 

### 메모리
메모리는 데이터를 저장하는 기억장치로, 메모리의 종류는 다음 사진과 같다. ![[Pasted image 20250913190510.png]]
주기억 장치는 프로세서가 수행할 프로그램과 데이터를 저장하며, DRAM을 주로 사용한다. 또한, 한 번에 너무 많은 프로그램이 디스크 작업을 요청해 생기는 현상인 디스크 I/O 병목현상을 완화할 수 있다. 

캐시는 프로세서 내부에 있는 메모리로, 속도가 빠르고 가격이 비싼 대신 CPU와 주기억장치 사이에서 생기는 메인 메모리 I/O 병목현상을 완화할 수 있다. ![[Pasted image 20250913191537.png]]
캐시에는 메모리 100번지를 보고 있을 경우, 그 근처의 메모리를 미리 올려 두는 방식으로 중요한 정보들을 올려 놓을 수 있으며, 필요한 데이터 블록이 캐시에 이미 있는 경우를 **Cache hit**, 없는 경우를 **Cache miss**라고 한다. ![[Pasted image 20250913191521.png]]

### 운영체제와 메모리
운영체제는 다음과 같은 작업들을 수행할 수 있다. 
![[Pasted image 20250913191812.png]]
- 메모리 할당 및 관리
	- 프로그램의 요청에 따른 메모리 할당 및 회수
	- 할당된 메모리 관리
- 가상 메모리 관리
	- 가상 메모리 생성 및 관리
	- 논리 주소 → 물리 주소 변환

### 시스템 버스 (System Bus)
시스템 버스란 컴퓨터 시스템 안에서 데이터나 메모리 주소, 제어 신호, 명령어 등을 주고받는 통로로, 다음과 같이 구성된다. 

- 주소 버스: CPU가 메모리 or I/O 장치에서 데이터를 읽고 쓸 때 사용하는 메모리 주소 전달
- 데이터 버스: 명령어나 데이터를 전달
- 제어 버스: CPU, 메모리, I/O 장치 간 제어 신호 전달
![[Pasted image 20250913192128.png]]

### 운영체제와 주변 장치
주변 장치도 운영체제의 영향을 받는다. 

- 장치드라이버 관리
	- 주변 장치(키보드, 마우스 등등) 사용을 위한 인터페이스 제공
- 인터럽트 처리
	- 주변 장치의 요청 처리
- 파일 및 디스크 관리
	- 각각의 운영체제마다 조금씩 다름 (Ext, NTFS, FAT 등등)
	- 파일 생성 및 삭제 관리
	- 디스크 공간 관리 등
