## HTML이란?
---
### HTML이란?
---
HyperText Markup Language (HTML)은 <u>웹 페이지를 구성하는 언어</u>이다. 사용자가 웹 브라우저를 통해서 웹 서핑을 하는 상황을 생각해 보자. URL 입력 칸에 웹 사이트의 주소를 입력하면 조금의 시간이 지난 후, 웹 페이지가 화면에 출력된다. 

![[Pasted image 20260105211658.png]]

예를 들어서 `google.com`에 접속하려고 하면, 브라우저는 `google.com`을 운영하는 웹 서버에 HTTP request를 보내며, `google.com`을 운영하는 서버는 해당 request를 받아 브라우저에게 웹 v페이지를 반환한다. 

사실, 여기서 웹 페이지라는 것은 결국 텍스트로 이루어진 정보이며, 이 텍스트에는 웹 페이지가 어떤 형태로 구성되며, 어떻게 동작할지가 정해진 규칙에 따라 작성되어 있다. 브라우저는 해당 규칙을 바탕으로 텍스트를 해석해, 화면에 웹 페이지를 띄운다. 

이때, 해당 텍스트는 HTML이라는 언어의 규칙을 따르며 HTML은 <u>웹 페이지를 구성하는 내용을 정의하고 그 구조를 만드는 데 사용하는 언어</u>로, 브라우저는 이를 읽고 해석해 시각적으로 화면에 보여 준다. 

물론, 실제로는 HTML과 더불어 CSS, Javascript 등 다양한 요소가 웹 페이지에 포함되어 있다.

## HTML 문법
---
### HTML 문법
---
HTML은 언어인 만큼, C나 Python처럼 특정한 문법이 존재하며, 이 문법에 따라 정해진 규칙대로 작성해야 한다. 기본적으로, HTML은 **요소(Element)** 들로 이루어져 있고, 이 요소들을 어떤 규칙대로 작성해야 하는지를 정해둔 것을 문법이라고 한다. 

HTML에서는 필수적으로 작성해야 하는 요소들이 있으며, 이번 페이지에서는 **HTML 요소가 어떤 구조를 띄는지** 살펴보고, **필수적으로 존재해야 하는 요소**들을 살펴본 뒤 **몇몇 주의해야 할 점**들을 공부한다. 


### HTML 요소
---
HTML 요소는 웹 페이지를 구성하는 기본 단위로, 웹 페이지에서 볼 수 있는 이미지, 텍스트, 하이퍼 링크, 상자, 입력 폼, 버튼 등이 HTML 요소라고 볼 수 있다. 

아래 사진은 HTML 소스 코드에서 각각의 HTML 요소들이 어떤 모습으로 웹 브라우저에서 표시되는지를 화살표의 형태로 매칭해서 보여 준다.

![[Pasted image 20260105213734.png]]

이번에는 하나의 HTML 요소의 구조를 알아보자. 
#### HTML 요소 구조 - 태그
```html
<태그이름>내용</태그이름>
```

하나의 HTML 요소는 여는 태그, 내용, 닫는 태그로 이루어져 있다. 

1. **여는 태그**: 소스 코드 상에서 `<`와`>` 사이에 태그의 이름이 포함된 `<태그이름>` 형태로 나타내며, 한 요소의 시작을 의미함
2. **내용**: 요소에 들어갈 내용
3. **닫는 태그**: 소스 코드 상에서 `</`와 `>` 사이에 태그의 이름이 포함된 `</태그이름>` 형태로 나타내며, 해당 요소의 끝을 의미함

HTML에서는 다양한 태그들을 지원하며, 아래와 같은 태그들이 대표적이다. "태그에는 이런 것들이 있구나" 하면서 한 번 읽어보고 넘어가면 된다. 

| 태그            | 설명                             | 예시 사용법                               |
| ------------- | ------------------------------ | ------------------------------------ |
| `<html>`      | HTML 문서의 루트 요소로, 모든 콘텐츠를 감쌈    | `<html> ... </html>`                 |
| `<head>`      | 문서의 메타데이터(제목, 스타일, 스크립트 등)를 포함 | `<head><title>페이지 제목</title></head>` |
| `<body>`      | 실제 화면에 표시되는 콘텐츠를 담는 영역         | `<body><h1>안녕하세요</h1></body>`        |
| `<h1> ~ <h6>` | 제목 태그, 숫자가 작을수록 큰 제목           | `<h1>메인 제목</h1><h2>부제목</h2>`         |
| `<p>`         | 문단을 나타내는 태그                    | `<p>이것은 문단입니다.</p>`                  |
| `<a>`         | 하이퍼링크를 생성하는 태그                 | `<a>링크 텍스트</a>`                      |
| `<img>`       | 이미지를 삽입하는 태그                   | `<img src="example_logo.jpg">`       |

#### HTML 요소 구조 - 속성
**속성(Attribute)** 은 태그에 추가적인 정보를 더하거나 특정한 동작을 지정하기 위해 사용한다. 아래는 속성이 존재하는 태그의 구조이다. 

```html
<태그이름 키1="값1" 키2="값2" 키3="값3">내용</태그이름>
```

속성은 위처럼 **키-값 쌍**의 형태로 작성되며, <u>여는 태그 안에 위치</u>한다. 또한, <u>여는 태그 안에서도 여러 개의 속성이 존재</u>할 수 있다. 


### HTML 요소 - 예시를 통해 이해하기
---
이번에는 HTML 요소가 어떻게 구성되어 있는지 실제 예제를 통해 자세하게 확인해 보자. 예제로 사용할 HTML 코드는 다음과 같다. 이 코드는 아까 실습으로 살펴 본 링크에서 하이퍼링크를 담당하는 요소이다. 

```html
<!--화면에는 안 보이게 숨기지만, 스크린 리더에는 읽히게 하는 CSS 클래스 -->
<!--접근성용 링크라는 의미로, 키보드나 스크린 리더 사용자가 본문으로 바로 이동-->
<a class="skip-link screen-reader-text" href="#content" title="Skip to content"> Skip to content</a>
```

1. 여는 태그 찾기
	1. 여는 태그는 `<태그이름>` 형태이므로, 위 요소에서 여는 태그는 `<a class="skip-link screen-reader-text" href="#content" title="Skip to content">`이다. 
	2. 여는 태그에는 키-값 쌍 형태로 속성이 존재할 수 있기 때문에, `class="skip-link screen-reader-text"`, `href="#content`와 `title="Skip to content"`라는 속성이 존재하는 것으로 해석된다. 
2. 닫는 태그 찾기
	1. 닫는 태그는 `</태그이름>` 형태이므로, 위 요소에서 닫는 태그는 `</a>`이다. 
3. 내용 찾기
	1. 내용은 여는 태그와 닫는 태그 사이에 존재하기 때문에, 위의 요소에서 내용은 `Skip to content`이다. 

간단하게 설명하면, `<a>` 태그는 **하이퍼 링크**를 만드는 태그로, 위 HTML 코드는 같은 페이지 안에서 id="content"를 가진 요소로 바로 이동(`href="#content"`)하며, 링크 위에 마우스 커서를 올렸을 때 "Skip to content"라는 툴팁 텍스트가 뜰 수 있도록(`title="Skip to content"`)한다. 
(`class=` 부분은 CSS 관련이기 때문에 이번 강의에서는 생략한다)

Dreamhack 강의에서는 `example.com`로 설명했지만, 나는 `videopsalm.org/download/`에 들어가서 예시를 찾았기 때문에, 간단하게 설명하기는 했어도 지금 이 단계에서는 어떤 태그와 어떤 속성이 무슨 기능과 무슨 역할을 하는지 완벽히 이해할 필요는 없다. 

중요한 태그와 속성은 이후 실습에서 만날 때마다 하나씩 설명할 것이므로, 그 외의 태그와 속성들은 실습을 통해 자연스럽게 접하면서 익히게 될 것이다. 


### HTML 요소 - 주의해야 할 점
---
HTML 요소를 다룰 때 주의해야 할 점들은 다음과 같다. 

1. **태그는 중첩 순서를 지켜야 한다**
	- HTML 요소는 서로 중첩해 사용할 수 있지만, 올바른 순서로 열고 닫아야 한다. 
	- 올바른 예시
		```html
		<p><strong>Emphasized text</strong></p>
		```
	- 잘못된 예시
		```html
		<p><strong>Emphasized text</p></strong>
		```

2. **속성은 여는 태그 안에 작성된다**
	- 예시: `href` 속성에 값이 `https://www.google.com`으로 설정되어 있다.
		```html
		<a href="https://www.google.com">Go to Google</a>
		```
	
3. **HTML은 대소문자를 가리지 않는다**
	- HTML 태그는 대소문자를 구분하지 않지만, 소문자로 작성하는 것이 편하다.
	- 권장하는 예시
		```html
		<p>This is a paragraph.</p>
		```
	- 권장하지 않는 예시 (사용은 해도 됨)
		```html
		<P>This is a paragraph.</P>
		```
	
4. **일부 태그는 닫는 태그가 없다**
	- HTML에서 지원하는 태그 중 일부는 닫는 태그가 존재하지 않기도 하다.
	- 스스로 닫히는 태그이며, 닫는 태그를 추가로 작성하지 않아도 된다. 
	- 닫는 태그가 없는 태그 예시
		```html
		<img src="dreamhack_logo.jpg">
		<input type="text" placeholder="Please enter your name.">
		```

### HTML 기본 문서 구조 공부하기
---
이번 장에서는 HTML 문서의 기본 구조를 살펴본다. HTML 문서는 다음과 같은 뼈대를 가진다. 

```html
<!DOCTYPE html>
<html>
	<head>
	  <title>My First Page</title>
	</head>
	<body>
	  <h1>Welcome to HTML!</h1>
	  <p>Hello, World!</p>
	</body>
</html>
```

#### `<!DOCTYPE html>`
1번 줄의 `<!DOCTYPE>`은 HTML 문서의 유형을 브라우저에게 알려 준다. 

해당 키워드는 **HTML 태그가 아니며**, 브라우저가 문서를 해석할 때 어떤 버전의 HTML을 기준으로 해야 하는지에 대한 정보를 제공한다.

#### `<html>`
2번 줄의 `<html>` 태그는 HTML 문서의 시작을 알려 주며, 10번 줄의 `</html>` 태그는 HTML 문서의 끝을 알려 준다. 모든 HTML 문서는 `<html>` 태그를 포함해야 한다. 

#### `<head>`
3번 줄의 `<head>` 태그는 HTML 문서의 메타데이터를 작성하는 부분으로, HTML5부터는 생략할 수 있다. 

해당 코드를 자세하게 살펴보면, 여는 태그와 닫는 태그 사이에 추가로 `<title>My First Page</title>`이 작성되어 있는데, 이는 웹 페이지 본문에는 표시되지 않지만, 다음 사진에서 빨간 네모로 표시된 것처럼 웹 페이지의 제목을 `My First Page`로 설정한다.

![[Pasted image 20260105230006.png]]
#### `<body>`
6번 줄의 `<body>`는 HTML 문서의 본문을 정의하는 부분으로, 웹 페이지에서 사용자가 직접 볼 수 있는 컨텐츠를 포함한다. 또한, **모든 HTML 문서는 `<body>` 태그를 포함해야 한다.**

코드를 보면 7~8번 줄에 `<h1>Welcome to HTML!</h1>`과 `<p>Hello, World!</p>`가 존재한다. `<h1>`은 본문에서 제목을 표시하며, `<p>`는 문단을 표시한다. 두 태그가 사용되어 위의 사진처럼 웹 페이지에 본문이 표시된다.

이번에는 HTML에 익숙해지기 위해 간단한 웹 페이지를 만들어 보고, 코드를 한 줄씩 해석해 보는 방식으로 공부한다. 실습에서는 우선 어떤 형태로 웹 페이지를 만들어야 하는지 결과물을 먼저 보여주며, 그 이후에 정답인 HTML 문서를 브라우저에서 확인한 뒤 코드를 해석해 본다. 

이번 실습에서 만들어 볼 페이지는 다음과 같다. 

![[Pasted image 20260105233457.png]]
**웹 페이지 설명**

- 브라우저의 탭 이름에서 보이듯, "My Simple Web Page"가 해당 웹 페이지의 제목이다.
- 제목은 "Explore the World of HTML!"이다.
- 내용은 "Greetings from the Web!"이다. 

먼저, 직접 웹 페이지를 만들어 본 뒤 아래에서 정답을 확인해 보자. 

```html
<!DOCTYPE html>
<html>
	<head>
		<title>My Simple Web Page</title>
	</head>
	<body>
		<h1>Explore the World of HTML!</h1>
		<p>Greetings from the Web!</p>
	</body>
</html>
```


### 정답 확인
---
정답 HTML 문서는 아래의 `simple_page.html`과 같다. 

```html
<!DOCTYPE html>
<html>
	<head>

	</head>
	<body>

```

결과를 위에서 직접 짠 것과 대조해 보면 알겠지만, 100% 일치하는 것을 확인할 수 있다.

### 코드 분석
---
이번에는 코드를 분석해 보자. 

• 1번 라인 -  `<!DOCTYPE html>`: HTML5 표준을 따르는 문서를 선언
• 2번 라인 - `<html>`: HTML 문서의 시작
• 3번 라인 - `<head>`: HTML 문서의 메타데이터를 작성하는 부분 
• 4번 라인 - `<title>~</title>`: 웹 페이지의 제목을 My Simple Web Page 로 설정 
• 5번 라인 - `</head>`: 3번 라인과 짝지어지며, `<head>` 요소를 닫는 태그
• 6번 라인 - `<body>`: HTML 문서의 본문을 작성하는 부분
• 7번 라인 - `<h1>~</h1>`: 헤딩(제목) 문구를 작성하는 태그
• 8번 라인 - `<p>~</p>`: 문단을 작성하는 태그이며, 내용은 Greetings from the Web! 
• 9번 라인 - `</body>`: 6번 라인과 짝지어지며, `<body>` 요소를 닫는 태그
• 10번 라인 - `</html>`: 2번 라인과 짝지어지며, `<html>` 요소를 닫는 태그

### 실습: 웹 페이지 만들기(심화)
---
이번 실습에서는 새로운 태그들을 사용해 복잡한 웹 페이지를 만들어 보면서 더욱 HTML에 익숙해 질 수 있도록 한다. 우리가 일상 생활에서 접속하는 웹 페이지들은 다른 페이지로 이동하는 링크나, 이미지 등 다양한 컨텐츠와 기능들을 제공한다. 

이번 실습에서 만들어 볼 페이지는 다음과 같다. 

![[Pasted image 20260105235005.png]]
**웹 페이지 설명**

- 브라우저의 탭 이름에서 알 수 있듯, 제목은 "Go to Dreamhack"이다.
- 내용에는 `<h1>` 태그로 만들어진 헤딩 문구 "Access dreamhack.io"가 존재한다. 
- 헤딩 문구 다음에는 `<img>` 태그로 만들어진 이미지가 표시되며, `dreamhack.png`이다. 
- 이미지 다음에는 `<a>`태그로, 링크 텍스트는 "Link to Dreamhack!", 링크 클릭 시 `https://dreamhack.io`로 이동한다. 
- 링크 다음에는 `<p>` 태그로 만들어진 "Hello, Dreamhack!"이라는 문단이 존재한다. 
- 그 다음에는 `<p>` 태그로 만들어진 "Dreamhack, the home for security experts"라는 텍스트가 존재한다. 

먼저 웹 페이지를 만들어 보는 게 권장되기는 하지만, 어렵다면 바로 정답을 확인해도 된다. 

먼저 웹 페이지를 만들어 보면 다음과 같다. 

```html
<!DOCTYPE html>
<html>
	<head>
		<title>HTML Tag Test</title>
	</head>
	<body>
		<h1>Access dreamhack.io</h1>
		<img src="./dreamhack.png">
		<a href="https://dreamhack.io">Link to Dreamhack!</a>
		<p>Hello, Dreamhack</p>
		<p>Dreamhack, the home for security experts</p>
	</body>
</html>
```


### 정답 확인
---
정답 HTML 문서는 다음과 같다.

```html
<!DOCTYPE html>
<html>
	<head>
		<title>HTML Tag Test</title>
	</head>
	<body>
		<h1>Access dreamhack.io</h1>
		<!--dreamhack.png: 정답 HTML 문서와 같은 디렉터리에 위치해야 함-->
		<img src="./dreamhack.png">
		<br> <!--직접 했던 것에는 없던 부분-->
		<a href="https://dreamhack.io">Link to Dreamhack!</a>
		<p>Hello, Dreamhack</p>
		<p>Dreamhack, the home for security experts</p>
	</body>
</html>
```


### 코드 분석
---
대부분의 내용이 앞과 비슷하니, `<br>` 태그와 `<img>` 태그, `<a>` 태그에 대해서만 알아보자. 

- `<br>` 태그: 내용에 개행(줄바꿈)을 추가하는 태그이다. 
- `<img>` 태그
	- HTML에서 이미지를 삽입할 때 사용하는 태그
	- 외부 이미지 파일을 불러와 웹 페이지에 표시할 수 있도록 도와줌
	- `src` 속성으로 불러올 파일의 경로를 지정할 수 있음
- `<a>` 태그
	- 하이퍼 링크를 삽입할 때 사용하는 태그
	- 웹 페이지 내에서 다른 웹 사이트, 외부 웹 사이트, 파일, 이메일 주소 등의 위치로 연결
	- `href` 속성으로 이동할 경로를 지정, 내용으로 링크의 이름을 지정
	- `<img>` 태그를 `<a>` 태그로 감싸면 이미지에 하이퍼 링크를 부여하는 것도 가능
		- 예시: `<a href="http://dreamhack.io"><img src="./dreamhack.png"></a>`


## 마치며
---
HTML의 기본 구조와 문법을 익히고, 간단한 웹 페이지를 제작하는 실습 또한 해 보았다. HTML은 모든 웹 페이지의 기초가 되는 언어로, 이를 제대로 이해하는 것은 웹 개발 뿐만 아니라 웹 해킹을 배우는 데 있어서도 중요하다. 

HTML의 기초를 잘 익혀두면 이후 다른 웹 프레임워크나 JavaScript 등의 지식을 배울 때도 훨씬 수월하게 학습할 수 있다. 또한, 웹 해킹을 배우면서 다양한 웹 페이지의 구조를 분석하고 이해하는 데에도 큰 도움이 된다.
