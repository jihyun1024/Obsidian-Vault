데이터 이동 방법은 다음과 같이 다섯 가지로 나눠진다. 

1. 값을 직접 레지스터로 대입
2. 레지스터에서 레지스터로 이동
3. 값을 직접 메모리로 대입
4. 레지스터에서 메모리로, 또는 그 반대로
5. 메모리에서 메모리로

x86에서는 모두 사용 가능하지만, ARM에서는 메모리에서 메모리로 직접 이동하는 것은 불가능하다. 

### mov dst, src 명령어

| mov 명령               | 설명                                              |
| -------------------- | ----------------------------------------------- |
| mov eax, ebx         | ebx 레지스터 내의 값을 eax 레지스터로 복사                     |
| mov eax, 0x42        | 값 0x42를 eax 레지스터로 복사                            |
| mov eax, [0x4037c4]  | 메모리 주소 0x4037c4의 값을 eax 레지스터로 복사                |
| mov eax, [ebx]       | ebx 레지스터가 명시한 메모리 주소에 있는 값을 eax 레지스터로 복사        |
| mov eax, [ebx+esi*4] | ebx+esi*4 연산 결과가 명시한 메모리 주소에 있는 값을 eax 레지스터로 복사 |
목적지로 값을 넣을 때 사용한다. 
값은 소스(src)에서 목적지(dst)로 이동하며, 위의 표는 mov 명령어를 사용한 예시를 보여준다.
src 부분의 대괄호로 묶인 값은 메모리 주소가 나타내는 값을 의미하며, src에는 연산이 필요한 수식은 들어갈 수 없다.

### lea dst, src 명령어

| lea와 mov의 차이     | 설명                             |
| ---------------- | ------------------------------ |
| lea eax, [ebx+8] | ebx+8을 eax에 저장                 |
| mov eax, [ebx+8] | ebx+8이 가리키는 주소에 저장된 값을 eax에 복사 |
lea는 **Load Effective Address**의 약자로, 메모리 주소 그 자체를 목적지에 넣을 때 사용한다. 
(C언어의 포인터와 비슷한 개념이다)

### push, pop 명령어
스택에 데이터를 넣거나 꺼내는 명령어로, push는 연산자의 값을 스택에 넣고, pop은 그 반대로 스택에서 값을 가져와 레지스터로 저장한다. 
이 명령어는 스택을 사용하기 때문에 스택 포인터인 esp의 값이 변경된다. 
 
| 명령어            | 설명                                 |
| -------------- | ---------------------------------- |
| pop eax        | 스택에서 값을 꺼내 eax에 저장 (esp가 4만큼 증가)   |
| mov eax, [esp] | 스택에서 값을 읽어와서 eax에 저장 (esp가 변하지 않음) |

추가적으로, 레지스터의 값을 스택에 일괄적으로 저장하기 위한 명령어가 있는데, 이는 작업 중인 내용을 일시에 저장하고 다른 작업을 수행하기 위한 것이다. 

| 명령어    | 설명                                 |
| ------ | ---------------------------------- |
| pusha  | 16bit 레지스터를 스택에 저장                 |
| pushad | 32bit 레지스터를 스택에 저장                 |
| pushf  | EFLAGS 레지스터의 하위 16bit를 스택에 저장      |
| pushfd | EFLAGS 레지스터를 스택에 저장                |
| pushfq | RFLAGS 레지스터를 스택에 저장                |
| popa   | 16bit 레지스터 정보를 스택에서 꺼냄             |
| popad  | 32bit 레지스터 정보를 스택에서 꺼냄             |
| popf   | EFLAGS 레지스터의 하위 16bit를 스택에서 꺼냄     |
| popfd  | EFLAGS 레지스터를 스택에서 꺼냄               |
| popfq  | RFLAGS 레지스터를 스택에서 꺼냄 (빈 곳은 0으로 채움) |
