### MCU란
Minimum Coded Unit의 약자로, JPEG 압축에서 인코딩과 디코딩의 최소 단위 묶음이다. 
각 MCU는 여러 컴포넌트(Y, Cb, Cr 등)의 8 x 8 [[DCT]] 블록들로 구성되며, SOF 마커의 sampling factor에 의해 MCU의 구조가 구성된다.

### MCU의 필요성
JPEG는 픽셀 단위가 아니라 8 x 8 DCT 블록 단위로 압축해 저장한다. 
이 경우 여러 컴포넌트를 다룰 때 각 컴포넌트가 서로 다른 해상도를 가지므로 한 덩어리 단위(MCU)로 처리해야 디코더가 더 잘 처리할 수 있다. 

### MCU 안의 블록 구성
각 컴포넌트가 MCU 안에서 차지하는 블록 수 = Hc x Vc
- Hc = 해당 컴포넌트의 H(Horizontal) 샘플링
- Vc = 해당 컴포넌트의 V(Vertical) 샘플링

**예시: 4:2:0 JPEG**
- Y: H=2, V=2 → MCU당 4블록
- Cb: H=1, V=1 → MCU당 1블록
- Cr: H=1, V=1 → MCU당 1블록
- ➝ 한 MCU = 총 6 블록 (= Y4 + Cb1 + Cr1)
- 픽셀 영역으로 보면:
    - MCU 크기 = (Hmax=2, Vmax=2) → 16×16 픽셀
    - Y는 16×16 전체 커버 (4 블록)
    - Cb, Cr은 각각 8×8 (샘플링 낮음, 색 정보 축소)

### JPEG의 인코딩과 디코딩
JPEG의 인코딩/디코딩은 MCU 단위로 스캔하며, 인코더는 이미지를 좌 > 우, 위 > 아래 순서로 MCU 별로 진행하고 디코더도 동일한 순서로 MCU를 읽고, MCU 내부의 컴포넌트 블록을 복원해 이미지를 조립한다. 

### 요약
- **MCU = JPEG의 최소 부호화 단위 = 여러 컴포넌트의 8×8 블록 묶음**
- 크기는 `Hmax×8 × Vmax×8` 픽셀
- MCU 안의 블록 개수 = Σ(Hc×Vc)
- 이미지 전체는 “MCU 격자”로 분할되어 부호화/복호화됨
