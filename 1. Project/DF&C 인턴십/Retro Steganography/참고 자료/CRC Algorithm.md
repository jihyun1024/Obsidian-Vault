
- CRC (Cyclic Redundancy Check) Algorithm
    
    데이터 전송이나 저장 중 일어날 수 있는 오류를 검출하기 위한 알고리즘으로, 모든 Chunk 끝에 붙어 데이터의 오류 여부를 확인한다.
    
    - CRC 계산 원리
        
        1. 데이터를 이진수로 간주한다.
        2. 생성 다항식으로 데이터를 나눈다.
        3. 데이터를 나눠서 얻은 나머지를 CRC 값으로 한다.
    - CRC 에서 사용하는 생성 다항식: 32차 다항식인 $x^{32} + x^{26} + x^{23} + x^{22}+x^{16}+x^{12}+x^{11}+x^{10}+x^{8}+x^{7}+x^5+x^4+x^2+x+1$ = 0x04C11DB7
        
    - CRC 계산 흐름
        
        ```c
        1. 초기값: CRC = 0xFFFFFFFF
        2. 각 바이트를 순서대로 처리:
            1. CRC = CRC ^ 각 바이트
            2. 8회 반복: CRC = (CRC >> 1) ^ (0xEDB88320 if (CRC & 1) else 0)
        3. 결과 = CRC ^ 0xFFFFFFFF
        ```
        
        |항목|설명|
        |---|---|
        |길이|32bit = 4byte|
        |초기값|0xFFFFFFFF|
        |다항식|0x04C11DB7|
        |후처리|계산 결과를 1의 보수 처리| 
	    
        - 8회 반복하는 이유: CRC는 bit 기반 알고리즘이므로 1byte를 bit 단위로 처리하기 위해 (1byte = 8bit)
        - 0x04C11DB7 대신 다른 식을 사용하는 이유: 32차 다항식은 최상위 비트 기준이고 위의 알고리즘은 최하위 비트 기준이기 때문에 0xEDB88320을 사용
