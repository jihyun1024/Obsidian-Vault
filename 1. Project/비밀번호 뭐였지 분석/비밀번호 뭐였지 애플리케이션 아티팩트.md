아티팩트: 소프트웨어 개발에서 생성되는 모든 산출물을 의미한다. 
예를 들어, 소스 코드, 문서 (요구사항 정의서, 설계서, 사용자 매뉴얼 등), 테스트 케이스 및 결과, 실행 파일 및 빌드 결과물, UI/UX 디자인 시안, 데이터베이스 등이 아티팩트에 해당한다.  
이 문서에서는 샘플 데이터와 sharedpreferences, 데이터베이스 등을 아티팩트로 판단하고 정리한다.

### 그룹, 공유 그룹 설정
* 일반 그룹
	* 사용자의 마음대로 항목을 분류해서 그룹화 할 수 있는 기능
* 공유 그룹
	* 프리미엄 멤버십에 가입되어 있지 않은 경우, 항목을 최대 10개까지만 추가할 수 있음
	* 실제로는 11개까지 추가 가능했음
	* 공유할 사용자를 초대하는 방법은 사용자의 계정 코드(앱 설정-계정)를 입력하여 초대할 수 있다. 
		* 이 때, 계정 코드와 리뉴얼 데이터 동기화 코드가 모두 == 기호로 끝난다. 
		* 따라서, 이 둘은 Base64 인코딩된 문자열일 가능성이 매우 높다는 것을 추측할 수 있다. 
### 2단계 인증 설정
가장 많이 사용하는 Google 기준으로 많이 복잡한 절차가 필요하다. 
그에 앞서, 구글 계정, 스마트폰, OTP 앱이 필요하다. 
여기서는 [Google Authenticator](https://play.google.com/store/apps/details?id=com.google.android.apps.authenticator2&hl=ko) 애플리케이션으로 OTP 앱을 사용했다. 
##### 1단계: 구글 계정 보안 설정 페이지 접속
웹 브라우저에서 https://myaccount.google.com/security 에 접속하여 아래로 내려가 **2단계 인증** 항목을 찾아 설정을 시작한다. 
##### 2단계: OTP 앱 추가
OTP 탭을 찾아 **+ 인증자 설정** 버튼을 클릭하면 다음과 같이 QR 코드가 뜨지만, 우리는 1개의 스마트폰으로만 하는 것을 전제하기 때문에 **스캔할 수 없나요?** 를 누르고 Google OTP 앱으로 들어간다. 
![[Pasted image 20250711011426.png]]

1. Google OTP 앱으로 들어가 + 버튼을 누르고 **QR 코드 스캔** 옵션 대신 **설정 키 입력** 옵션을 선택한다. 
2. 코드명은 맘대로 해도 좋고, **내 키**에는 방금 전의 알파벳과 숫자가 결합된 32자리 문자열을 대입한다. 
3. 키 유형에서 **시간 기준**을 선택했는지 확인한다. 
4. 밑의 **추가** 버튼을 눌러서 완료한다. 
![[Pasted image 20250711011444.png]]
##### 3단계: 비밀번호 뭐였지? 에 등록
이제는 Google에 2단계 인증이 등록된 상태이기 때문에, 컴퓨터에서도 이를 확인할 수 있다. 
똑같이 Google 계정 - 보안 - 2단계 인증 - OTP에 들어오면 OTP가 추가된 것을 확인할 수 있고, **인증자 앱 변경**을 클릭하면 QR코드 또한 확인할 수 있다.![[스크린샷 2025-07-11 002859.png]]

* 해당 계정에 대한 로그인 시도 시, OTP 앱에서 생성된 6자리 코드의 입력이 요구된다. 
* 30초마다 코드가 갱신된다.

이를 비밀번호 뭐였지? 에 등록하기 위해 비밀번호 뭐였지 앱을 실행, 2단계 인증 탭에 들어가 **QR 코드 스캔**을 선택하고 해당 QR 코드를 스캔하면 2단계 인증 탭에 해당 정보가 저장되며, 정보를 클릭하면 자동으로 6자리 코드가 클립보드에 복사된다. 
![[Pasted image 20250711011554.png]]

따라서, 2단계 인증 기능은 google이나 다른 여러 사이트에서 로그인할 때 기존의 문자로 받던 6자리 코드 대신 사용자의 디바이스에서 자동으로 6자리 코드를 생성하여 이를 바탕으로 보안을 하는 것이다.

이 중 주목할 파일이 들어있는 디렉터리를 정리하면 다음과 같다. 
* databases: 원래대로라면 데이터베이스가 들어 있어야 하나, 그 대신 EVENTS 파일이 데이터베이스에 쓰이는 SHM, WAL 파일과 같이 저장되어 있음
	* com.google.android.datatransport.events
	* com.google.android.datatransport.events-shm
	* com.google.android.datatransport.events-wal
* no_backup: databases 디렉터리와 마찬가지로 SHM, WAL 파일이 저장되어 있음
	* androidx.work.workdb
	* androidx.work.workdb-shm
	* androidx.work.workdb-wal
	* com.google.android.gms.appid-no-backup
* shared_prefs
	* admob.xml
	* AwOriginVisitLoggerPrefs.xml
	* com.google.android.gms.appid.xml
	* com.google.android.gms.signin.xml
	* paid_storage_sp.xml
