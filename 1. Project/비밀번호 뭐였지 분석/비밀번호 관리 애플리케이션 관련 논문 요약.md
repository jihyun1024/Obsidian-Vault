1. 로컬 기반: Password Safe and Manager, Enpass처럼 비밀번호 데이터를 사용자의 기기 내에 저장
2. 클라우드 기반: LastPass, 1Password처럼 비밀번호 데이터를 중앙 서버 또는 클라우드에 저장


### 논문 요약
김한결, 이신영, 박명서. (2024). 비밀번호 관리 어플리케이션의 주요 데이터 복호화 연구 및 보안성 평가. 정보보호학회논문지, 34(1), 61-70
##### 분석 대상 애플리케이션
논문에서는 Password Cloud, Password Management, HidePass, BeEasy 4가지의 애플리케이션을 분석하였으며, 현재 Google PlayStore에서는 Password Management, BeEasy를 찾아볼 수 없었음

##### Password Cloud
* **비밀번호 저장 방식**: 마스터 패스워드를 AES256/ECB/PKCS5Padding으로 암호화하여 관리하며, 모든 사용자의 데이터는 Temp.db 파일에 평문으로 저장되었음
* **특징**: 사용자가 양식을 커스텀하여 저장할 수 있는 기능을 제공
* **보안성**: 잠금 해제 시도 실패가 여러 번 지속될 시 모든 모듈을 삭제하는 기능이 있고, 사용자의 데이터를 암호화하지 않음

##### Password Management
* **비밀번호 저장 방식**: 마스터 패스워드는 sharedPreference에 암호화되어 저장되며, 사용자의 데이터는데이터베이스의 safe 테이블에 암호화되어 저장되었음
* **특징**: 데이터들 중 웹 사이트의 계정 정보만을 기록하며, 마스터 패스워드는 SHA256을 통해 생성
* **보안성**: 해시 테이블에 대응하기 위해 해시값 생성 이전에 임의의 문자열을 추가하며, 사용자의 데이터를 암호화할 때 IV와 암호화에 사용한 키는 하드코딩된 값을 바탕으로 AES128/CBC/PKCS5Padding으 암호화

##### BeEasy
* **비밀번호 저장 방식**: 마스터 패스워드는 sharedPreference에 암호화되어 저장되고, 사용자의 데이터 및 이미지 데이터는 write.db 파일에 암호화되어 저장
* **특징**: 텍스트 뿐만 아니라 이미지도 저장 가능하며, 마스터 패스워드를 6자리 PIN으로 설정
* **보안성**: 마스터 패스워드의 경우 숫자만 사용하기 때문에 약 1.7초 내의 빠른 시간 내에 모든 경우의 수를 확인할 수 있고, 사용자의 데이터를 암호화하는 키와 IV가 하드코딩되어 있음

##### 결론
1. 복호화 과정
	1. 마스터 패스워드, 저장 위치, 암호화 알고리즘에 대하여 파악할 수 있었음
	2. 대칭키 암호화의 경우, 동일한 알고리즘과 파라미터를 사용하여 복호화 할 수 있음
	3. 해시함수를 통한 암호화의 경우, 전수조사 방식으로 복호화 시도를 할 수 있음
2. 보안성 평가 및 대응 방안
	1. 키 생성: 하드코딩된 키를 사용하지 말고, 사용자의 환경별로 다른 암호화 키를 생성할 필요
	2. 대칭키 암호화: 암호화에 사용되는 키의 안전한 저장이 필요함
	3. 해시함수를 통한 암호화: 암호학적 salt를 추가하여 해시 테이블에 대응할 수 있으며, 패스워드 설정 시 강력한 패스워드를 요구하여 무차별 대입 공격에 대응할 필요
