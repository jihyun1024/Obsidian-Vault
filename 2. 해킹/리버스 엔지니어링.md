##  1. 정의
리버스 엔지니어링은 **완성된 시스템이나 프로그램을 분석하여 그 내부 구조, 동작 방식, 설계 원리 등을 역으로 추론하는 기술**이다. 
주로 **소프트웨어 바이너리 분석**, **악성코드 분석**, **보안 취약점 탐지**, **프로토콜 분석** 등에 활용된다.

---
##  2. 목적

- **악성코드 분석**: 실행 파일의 내부 동작을 파악하여 감염 방식, C2 통신, 암호화 방식 등을 분석
- **취약점 분석**: 소프트웨어의 보안 결함을 찾아내고 익스플로잇 가능성을 평가
- **라이선스 우회 탐지**: 불법 크랙이나 패치 여부 확인
- **프로토콜 분석**: 문서화되지 않은 통신 프로토콜을 역설계하여 이해
- **디지털 포렌식**: 침해 사고 발생 시 실행 파일의 행위 분석을 통해 증거 확보

---
##  3. 주요 대상

- 실행 파일 (.exe, .dll, .so 등)
- 스크립트 파일 (Python, JavaScript 등)
- 펌웨어 및 임베디드 시스템
- 네트워크 트래픽에서 추출된 바이너리

---
## 4. 주요 기술

### 🔧 정적 분석 (Static Analysis)

- 실행하지 않고 코드 구조를 분석
- 도구: [[Ghidra]], IDA Pro, Binary Ninja
- 분석 대상: 함수 구조, API 호출, 문자열, 암호화 루틴

###  🔥 동적 분석 (Dynamic Analysis)

- 실제로 실행하여 동작을 관찰
- 도구: [[x64dbg]], OllyDbg, Process Monitor, [[GDB]]
- 분석 대상: 메모리 변화, 파일 생성, 네트워크 통신

###  🧠 디컴파일 & 디스어셈블

- **디컴파일**: 바이너리를 고급 언어로 변환 (예: C)
	- 도구: Jadx, [[Ghidra]]
- **디스어셈블**: 바이너리를 어셈블리 코드로 변환

---
##  5. 분석 절차 예시

1. **샘플 확보**: 악성코드 또는 분석 대상 바이너리 수집
2. **정적 분석**: 문자열, 함수 구조, 의심 API 확인
3. **동적 분석**: 실행 후 행위 관찰, 로그 기록
4. **행위 분석**: C2 통신, 파일 생성, 레지스트리 변경 등
5. **보고서 작성**: 분석 결과 문서화 및 대응 방안 제시

---
##  6. 학습 팁

- CTF 문제를 통해 실전 감각 익히기
- 악성코드 샘플 분석 실습 병행 ([[Ragnar Locker 랜섬웨어 분석]], [[랜섬웨어 분석 (2025)]] 등)

- 어셈블리 언어와 운영체제 구조에 대한 이해 필수
